1. 블록체인


① 블록: 정보를 DB에 추가하는 방법(Bitcoin의 경우 10분 마다 '블록'이 생성됨)
  • hash of the block : 일방향 함수, 결정론
  • previous hash
  • data(ex: 결혼증명서, 운전면허증, 대학교 학위, .... ) bitcoin의 경우 '거래내역(transaction)'
② 특징
  • 오직 추가만 가능
  • 탈중앙화
if 블록이 비트코인 블록체인에 추가되고 싶다면?
bitcoin의 결제내역 모으기 + 이전 블록의 해시 → 나만의 해시 탄생

2. 어떤 데이터가 블록체인에 추가될 수 있는가?


오직 '진실'인 데이터만 필요함 이를 알기 위해서는 '작업증명(Proof of Work)'방식을 이용한다
① 작업증명

check ⟹ input ⟹ send

채굴자들은 블록체인에 들어오는 데이터가 '진실'인지 확인하고 블록에 넣어서 블록체인에 올리는 일을 한다.
즉 공공거래장부 검증에 참여해 암호화폐로 보상을 받는다.
이는 탈중앙화 방법으로 진행되기 때문에 원한다면 누구나 채굴자가 될 수 있다.
하지만 블록체인에 올리는 방법은 매우 어려움


블록을 올릴 때마다 처음엔 50개의 비트코인이 생성되었으나 4년마다 반감기가 와서 반틈씩 줄어듬(현재는 6.25개/비트코인은 2100만개로 생산량 한정됨)

3. 비트코인이 지루한 이유


: 오직 bitcoin을 주고 받는 것만 가능하고 다른 사람과 교류할 수 없음

4. 스마트컨트랙트

: 내가 작성한 코드를 모두가 공유, 검증, 실행할 수 있지만 수정은 할 수 없는 백엔드에 올린다. 비트코인 시스템과 다르게 다른 사람과 교류가 가능하며 사람과 로봇이 소통이 가능하다.
이를 통해서 탈중앙화 앱을 만들 수 있음.

ex) airbnb가 하는 일을 smart contract가 대신한다면?
: 고객이 smart contract에 돈을 보내면 IOT기술을 활용해서 대문이나 TV같은 물체에 연결해 문을 열어줄 수 있다.

smart contract를 지원하는 블록체인은 매우 많은데 이더리움이 가장 유명하다. 그 외에도 Kusama, Polkadot, Cardano, Cosmos 등이 있다.

5. NFT(Non fungible Token)

① 토큰이란 무엇인가?
: 이더리움 같은 블록체인은 나만의 토큰을 발행할 수 있다. 즉 블록체인의 안정성을 기반으로 화폐를 발행해서 마음대로 활용할 수 있음.

② 어떻게 토큰을 발행하는가?
: 2가지 기능(돈을 받으면, 돈을 보내준)을 갖고 있는 스마트 컨트랙트를 만든다.
돈을 받으면 딱 1개만 발행되는 토큰을 만든다면? 디지털 자산의 소유권에 대한 정보를 담을 수 있다.= NFT

③ 왜 돈을 주고 NFT를 사는가?
자신이 갖고 있는게 창작자가 만든 원본임을 증명해주기 때문에

6. 지갑

① 비트코인을 소유한다는게 무슨 의미일까?
: 내가 집을 샀다고 해서 그 집의 주소가 바뀌지 않듯 비트코인을 보낸다= 내 소유로 등록된 비트코인을 다른 사람의 소유로 등록한다

② 비대칭적 암호화(asymmetric encryption)
: 비트코인 보유자는 공개키(자물쇠)와 비공개키(열쇠)를 가진다.
즉 지갑을 생성한다 = 비공개 키를 만든다. 여기서 만든 비공개키로 공개키(아주 긴 텍스트)를 만든다. 공개키는 다른 사람에게 비트코인을 보낼 때 공유해야 할 주소이다.

7. 금융의 미래 DeFi

정의:Decentralized + Financial
stable coin: 암호화폐 중 하나 but USD와 1:1로 연결되어 있고 주로 회사가 보증을 한다. USDT, USDC, BUSD(바이낸스가 보장) … 있다.
여기에 스마트 컨트랙트를 사용할 수 있다는 점이 합쳐지면서 많은 걸 할 수 있음.

그 중 하나가 "AAVE"

① AAVE
: 유동성 풀(liquidity pool)(돈을 빌리고 빌려주는 은행같은 것)(현재 10조원의 돈을 관리하고 있음)
여기서 USDT, BUSD 같은 걸 빌려주고 연 9~12% 이자를 벌 수 있음.

② DEX *탈중앙화 거래소*
: 유니스왑, 스시스왑, 팬케이크 스왑은 사람의 개입없이 100% 코드로만 돌아간다.
여기에선 사용자들이 유동성(자산을 현금으로 전환할 수 있는 정도를 나타냄)을 제공하고 그에 대한 보상으로 거래 수수료를 받는다.
사용자들이 중간 거래자 없이 서로 토큰을 주고 받을 수 있음

ex) 만약 YFI 토큰과 LINK 토큰을 바꾸고 싶다면?
YFI토큰과 LINK토큰 유동성풀에 간다 - YFI 토큰을 넣으면 스마트 컨트랙트를 통해 LINK 토큰이 나에게 온다.
ex) 내가 1000불을 투자하고 싶다면? 500개의 스테이블 코인을 사고 그 다음 500불치의 ETH을 산다 ⇨ BUSD-ETH 유동성 풀에 넣는다.

③ 탈중앙화 카지노에서 베팅
: 현재 스테이블 코인(USDT, USDC, BUSD)을 이용해서 도박을 할 수도 있음.

8. PoS(Proof of stake)

: 작업증명의 단점(엄청난 양의 전력과 컴퓨터 파워를 낭비하고 보상만 바라고 비트코인 프록젝트에 대한 믿음도 없이 해당 네트워크를 보호하려는 점)을 보완하기 위해 PoS가 나왔다
PoS에서 검증자, 혹은 채굴자가 되는 방법은 내가 보호하려는 프로젝트의 코인을 소유하고 staking(네트워크에 고정적으로 맡김)이다.
한계점: 작업증명이 빠른 컴퓨터에게 보상하듯 지분증명은 "돈 많은 사람"에게 보상함. 이를 해결하기 위해 Delegated PoS(위임지분증명)(but 아직까지는 작업증명방식만큼 안정성이 확보되지 않은 상태)이 생김
